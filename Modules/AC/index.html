Ontology for Appraisal Costs (OCQA-AC)

The OCQA-AC ontology is an extension of the core Ontology for Construction Quality Assurance (OCQA), itself built upon the DiCon ontology for construction workflows. OCQA-AC adds the necessary structures to represent and estimate appraisal costs for quality inspections. It reuses the entire OCQA (and DiCon) via owl:imports, ensuring full alignment and hard reuse of existing concepts.

OCQA-AC leverages the inspection planning rules already defined in OCQA and introduces cost‐related information—such as labor rates, equipment usage fees, material costs, and overhead—so that every inspection instance can be automatically linked to its relevant cost data.

Appraisal Cost Estimation

Appraisal costs (ACs) are the expenses incurred to verify that construction work meets the required quality standards. These costs typically include both overhead expenses and variable expenses linked to each inspection carried out.

Overhead expenses (CX,O) cover items such as travel to inspection sites, setup and teardown of testing equipment, and any external acceptance fees. Variable expenses depend on the number of inspections (n) and consist of:

  • Labor costs for inspectors (CX,L)
  • Material and consumable costs (CX,M)

The general formula to estimate appraisal costs for a given quality activity X is:

ACX = CX,O + n × (CX,L + CX,M)

In practice, estimating ACs involves three main steps:

  1. Document the current quality assurance expenditures across all projects.
  2. Categorize each cost item into labor, equipment setup, and materials or consumables.
  3. Apply the formula above to calculate total appraisal costs and, if needed, adjust procedures to optimize individual cost components.

Note that appraisal cost estimation can be adapted to fit any organization’s internal guidelines or industry standards (e.g., ISO 9001 or DIN 55350). Depending on the chosen inspection strategy—whether 100% inspection or sampling—overall ACs will vary accordingly.

Cost Data Catalog

All cost indices and rates are stored in a DCAT‐compliant catalog structure with a main catalog and subcatalogs (See Figure 2). Each catalog entry holds a numeric cost value (e.g., hourly rates, per‐inspection fees, travel and setup costs) that can be updated via the OPM ontology. Inspections, equipment, and personnel are linked to these cost entries through simple properties, allowing the ontology to reference up‐to‐date cost data during inference.

SHACL Rules for Cost Inference

Appraisal‐cost estimation in OCQA-AC is performed entirely within the ontology using SHACL in two phases:

  1. Association: OWL restrictions automatically assign each inspection, its equipment, and personnel to the appropriate cost entries from the catalog.
  2. Computation: Two SHACL shapes execute SPARQL‐based rules:
    • The first multiplies the number of inspection events by their per‐inspection cost values.
    • The second aggregates all individual cost components (labor, material, overhead) into a single appraisal‐cost property on each inspection.

# Example SHACL rules for appraisal cost inference

# 1. Calculate per‐inspection costs
sh:NodeShape a sh:NodeShape ;
  sh:targetClass ocqa:Inspection ;
  sh:rule [
    a sh:TripleRule ;
    sh:subject sh:this ;
    sh:predicate ocqa:hasEquipmentCost ;
    sh:object [
      ocqa_rule:multiply ([ sh:path ocqa:hasQuantity ]
                          [ sh:path ocqa:hasInspectionEquipment/schema:value ]) ;
    ] ;
    sh:condition ocqa_rules:costEquipmentCondition ;
  ] .

# 2. Sum all cost components
sh:NodeShape a sh:NodeShape ;
  sh:targetClass ocqa:Inspection ;
  sh:rule [
    a sh:SPARQLRule ;
    sh:construct """
      PREFIX ocqa: <...>
      CONSTRUCT { $this ocqa:hasAppraisalCost ?total }
      WHERE {
        SELECT $this (SUM(?cost) AS ?total)
        WHERE {
          $this ocqa:hasEquipmentCost ?cost .
          # include other cost values here
        }
        GROUP BY $this
      }
    """ ;
  ] .

This declarative, standards‐based approach enables domain experts to maintain and adapt cost‐calculation logic directly in the ontology, without external scripts.

The SHACL shapes for OCQA-AC are published at https://SebSeis.github.io/OCQA/Modules/AC/InferenceRule_Inspection_Costs.ttl and can be validated against your RDF with the pySHACL runner script available at https://SebSeis.github.io/OCQA/Modules/AC/run_shacl_ac_inference.py (e.g., pyshacl --shacl shapes/ocqa-ac-shapes.ttl --data data/ocqa-ac-data.ttl --output results/ocqa-ac-results.ttl).
