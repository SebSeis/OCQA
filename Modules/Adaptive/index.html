Overview

To represent different states of inspection plans and related information, OCQA-Adaptive uses contextualization. The framework defines contexts to structure inspection-related data and support adaptive planning of quality inspections during construction. Instead of overwriting values, evolving information is represented in contexts so that different states can be tracked, compared, and reused. Context can be represented using qualified properties, objectified properties, or named RDF graphs. While an ontology provides a shared vocabulary, context specifies the conditions under which that vocabulary is applied, allowing conflicting representations to coexist in one knowledge base and enabling systematic comparison and reasoning.

OCQA-Adaptive follows the FAIR principles and is accessible at https://w3id.org/ocqa-adaptive#. Contextual information is represented using named graphs due to high performance and reduced complexity. Named graphs support versioning and status tracking of entities such as inspections and building objects. Each named graph is linked via ocqa-adaptive:hasContent to an instance of ocqa-adaptive:Context that captures version or status metadata. Version-related named graphs capture releases and changes to information models; micro changes can be recorded within the named graph using qualified or objectified properties.

In parallel, OCQA-Adaptive provides an orthogonal layer that represents the current status of project entities using four contexts: As-Designed, As-Built, As-Planned, and As-Performed. By linking contexts with relations such as ocqa-adaptive:dependsOn and ocqa-adaptive:nextContext, the framework supports adaptive inspection planning across the construction lifecycle and enables comparisons between planned and performed inspections and between design and construction states. Contexts can also be grouped using ocqa-adaptive:ContextCollection, for example by spatial, temporal, or authorization status, supporting transparent long-term tracking and dynamic adaptation.
